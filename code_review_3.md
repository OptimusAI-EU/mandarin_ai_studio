#Mandarin AI Studio
## Backend
-  Models page is loading models from openrouter correctly and categorizing them correctly. However, the model cards do not display the input and outpricing correctly and just show $0.0/1M token for all models. Pricing data should be retrieved for input and output tokens during sync and displayed correctly on the model card.
- Include all models with the input_modalities as "text" and "image" and output_modalities as "text" should be merged with the Image category.
- On the models page, if a model is selected, by clicking the "Select this Model" button, it should only be available to its respective category on the frontend. For example, if a model is selected from "text" category, it should only be available to the "text" module on the frontend and not across the board.
- There should be a dedicated search bar to search for models in addition to the existing dropdown.
- The "Sessions" API should count and store as sessions the entire conversation (up to that point) in the database under respective categories i.e. text, image, video, audio etc., and not merely each turn in the conversation. That is why sessions should be managed with IDs. Clicking on a particular session should open it in the main chat module for its respective category, for example, image sessions should be opened only in the image module in the UI and should not be available across the board. These turns in conversation are presently being opened in the Preview pane in the UI, which should not be the case, sessions should load in the main widget as a complete thread with the ability to continue conversation from there onwards.
- User should be able to delete a session from the database by clicking on a delete button.
- The "Artifacts" API should not display conversations as it is presently doing. Artifacts are complete code snippets generated during chat, images, documents, digital assets, videos etc. These artifacts should be displayed here with the option to open and save. If save option is exercised, these should be saved to the database with cross-reference to session ID etc.
## Frontend
- If you start a conversation in one module, for example "Text", it should be available and visible only in that module and not spill over to other modules,for instance "Image", "Audio" etc. That is why it is important to categorize each conversation as a session under its respective category. Modules/categories should be isolated from each other. 
- To start a new conversation, a New Session option should be provided in the side bar, which should enable a new chat with its own session ID under the respective ID.
- Presently, the chat is spilling over to the right sidebar preview pane. Each conversation is repeated there, which should not be the case.
- The "upload attachments" icon should enable uploading "files", "images", "folders", "audio" and "video" to the context of the conversation. User should also be able to just copy and paste an image in the conversation. Presently, it is not providing all these options. Moreover, the options which are available i.e. upload file/image are not function. It should be clearly visible in the chat widget that what has been attached or pasted in the context of the conversation.
- There should be a stop button to intercept and stop the conversation at that point. Use standard stop icon to display the button
- There should be a "regenerate response" button at the end of each response from the model to regenerate the respose. Use standard refresh icon to display the button.
