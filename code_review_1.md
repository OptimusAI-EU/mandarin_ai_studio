#Mandarin AI Studio
## Backend
Backend is almost completely broken after this update. Following needs to be addressed:
- A new module named text/multimodal should be included in addition to the existing ones. Its primary purpose will be coding, translation, and other text/image/multimodal related tasks in accordance with model capabilities.
- The Model page does not retrieve a complete list of models from the openrouter API endpoint https://openrouter.ai/api/v1/models. It currently displays only 13 video models. A complete list should be retrieved whenever the models link in the UI is clicked and then parsed into various input modalities such as text, image, video, multimodal etc. The dropdown on the models page is also non-functional, it should be able to display the models according to various input modalities. All models must be displayed with their price and context and clicking on a model should open its link to the openrouter site.
- The settings page is completely dysfunctional. The save button for API keys should work properly and the API keys should be moved to a .env file once the save button is pressed. The test button on this page is also dysfunctional and should be able to test the openrouter endpoint and display the result of testing the endpoint. Both the settings and .env should be included in the .gitignore.
- UI
The UI is completely dysfunctional. Following needs to be addressed:
Include a new link to cover text/multimodal mode in the Navbar. All coding operations done here must lead to creation of an artifact on the right side panel displaying the code with the option to save as well as execute. Execute must lead to opening of a browser tab or a REPL according to the requirement. All other text or image based operation must be performed inline.
The model dropdown for none of the modules works. It should be populated with the relevant models for a particular input modality as soon as the drop down is clicked. For example if you are working in the image module, the drop down should be populated with image models by calling the openrouter API endpoint. Similarly for all the other modules as well including text, video, 3D and audio. Only relevant models for that particular mode should populate the dropdown not models meant for other modules/input modalities.
The attachment icon in the UI does not attach files, folders, images, urls or anything else to the conversation. Its functionality needs to be corrected.
